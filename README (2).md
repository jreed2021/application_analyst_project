# Application Analyst Simulation Project
## Dual-Track Interview Prep for Karchem Consulting and OCHIN

This project simulates daily application analyst work across **two different environments**:

- **Karchem Consulting**: laboratory informatics consulting focused on ELN/LIMS support, requirements gathering, configuration, automation, training, onboarding/offboarding, and issue resolution for biotech clients.
- **OCHIN**: healthcare IT support focused on **Epic Willow Ambulatory**, workflow analysis, system build, testing, documentation, and end-user support in a large remote network serving community healthcare organizations.

The goal is to help you prepare for interviews by practicing the exact type of analyst work both roles emphasize, while also learning how to speak clearly about the **differences** between laboratory systems support and healthcare EHR/pharmacy application support.

---

## Project Goals

By the end of this project, you should be able to:

1. Explain how an application analyst works through a request from intake to resolution.
2. Show how requirements, configuration, testing, documentation, and training fit together.
3. Speak separately about:
   - **Lab informatics workflows** for Karchem
   - **Epic Willow Ambulatory workflows** for OCHIN
4. Demonstrate that you understand how company mission affects analyst priorities.
5. Walk through realistic tickets during interviews without sounding generic.

---

## Why This Project Fits Both Roles

### Shared analyst work across both jobs
Both roles involve:
- understanding user workflows
- documenting requirements
- making controlled application changes
- testing builds/configuration
- troubleshooting user issues
- creating training or support documentation
- collaborating with technical and operational stakeholders

### What makes Karchem different
Karchem sits closer to **biotech R&D and lab operations**. Their role emphasizes:
- ELN/LIMS and related lab systems
- scientific workflow understanding
- client-facing consulting
- vendor/platform variation
- configuration through UI and API-based workflows
- onboarding/offboarding, managed services, and lab-specific optimization

### What makes OCHIN different
OCHIN sits closer to **community healthcare delivery**. Their role emphasizes:
- Epic EHR application support
- Epic Willow Ambulatory workflows
- healthcare operations and patient-facing impact
- build, testing, patches, and validation
- member/end-user support
- learning under experienced analysts in a structured apprenticeship model
